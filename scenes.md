# 相遇问题：两个人什么时候碰面？

## Overview
- **Topic**: 小学数学相遇问题 — "速度和 × 时间 = 总路程"
- **Hook**: 小明和小红同时从家出发，相向而行，什么时候会相遇？
- **Target Audience**: 小学四~五年级学生，已掌握基本四则运算
- **Estimated Length**: ~4 分钟
- **Key Insight**: 两种解法殊途同归 — "分别算再加" 和 "速度和×时间" 其实是乘法分配律的两种面貌

## Narrative Arc
从一个生活化场景出发（两个小朋友相向而行），用线段图把抽象问题视觉化，然后展示两种解法。最后揭示两种解法之间的数学联系（乘法分配律），让观众感受到"算得巧"背后的道理。

---

## Scene 1: 问题引入 — 两个小朋友的故事
**Duration**: ~40s
**Purpose**: 用生活化场景建立问题，让观众产生好奇心

### Visual Elements
- 一条水平道路（直线），两端各有一个卡通小人图标
- 左侧标注"小明家"，右侧标注"小红家"
- 道路上方显示距离标签：`840米`
- 两个小人图标分别标注速度：小明 `70米/分`(BLUE)，小红 `50米/分`(GREEN)
- 两个箭头指向中间，表示"相向而行"

### Content
1. 屏幕上出现一条道路，两端各有房屋图标
2. 小明（蓝色小人）出现在左端，小红（绿色小人）出现在右端
3. 标注两地距离 840米
4. 两人的速度分别出现在各自上方
5. 两个方向箭头从两人指向中间，伴随文字"同时出发，相向而行"
6. 画面定格，出现问号和大字：**"几分钟后相遇？"**

### Narration Notes
- 语气：讲故事般的亲切感
- "小明和小红约好了一起玩。小明从家出发，每分钟走70米；小红也从家出发，每分钟走50米。"
- "两家相距840米。两个人同时出发，朝着对方走——那么，几分钟后他们会碰面呢？"

### Technical Notes
- 使用 `Line` 画道路
- 使用 `Dot` 或小圆形 + `Text` 表示人物
- `Arrow` 表示方向
- `Tex` 显示速度和距离

## Scene 2: 画线段图 — 把问题"画"出来
**Duration**: ~70s
**Purpose**: 用线段图建立视觉模型，这是理解问题的关键一步

### Visual Elements
- 一条水平线段，两端标记为 A（小明家）和 B（小红家）
- 线段总长标注 `840米`
- 线段上出现一个可移动的相遇点标记（初始在中间偏右位置，因为小明更快）
- 相遇点将线段分成两段：
  - 左侧蓝色段 = 小明的路程
  - 右侧绿色段 = 小红的路程
- 标注：蓝色段下方 `70 × ?`，绿色段下方 `50 × ?`
- 核心等式出现在线段图下方：
  ```
  小明的路程 + 小红的路程 = 总路程
     70×?    +    50×?    =  840
  ```

### Content
1. 道路切换为抽象线段图（保持空间布局一致）
2. 线段的两个端点出现 A、B 标记
3. 相遇点从左侧开始向右移动（示意两人运动），在线段上某处停下
4. 线段被分成蓝色段和绿色段
5. 蓝色段标注"小明的路程 = 70 × 时间"
6. 绿色段标注"小红的路程 = 50 × 时间"
7. 等式从两侧汇聚到中央，逐步显示

### Narration Notes
- 语速放慢，给学生思考时间
- "让我们画一条线段。A点是小明家，B点是小红家，全长840米。"
- "两人在中间的某个位置相遇——小明走得快，所以相遇点离小红家更近一些。"
- "你看，小明的路程是70乘时间，小红的路程是50乘时间，两段加起来就是840米。"
- 停顿——"这就是我们的等量关系。"

### Technical Notes
- `NumberLine` 或自定义 `Line` + 刻度标记
- `Dot` 作为相遇点，用动画沿线段移动
- `Brace` 标注各段距离
- 等式用 `MathTex`，逐步 `Write` / `FadeIn`
- 颜色：小明相关用 BLUE (#58C4DD)，小红相关用 GREEN (#83C167)

## Scene 3: 解法一 — 分别算，再加起来
**Duration**: ~50s
**Purpose**: 展示第一种解法（也是学生最直觉的方法）

### Visual Elements
- 左侧保留简化线段图
- 右侧出现解方程过程：
  ```
  解：设 x 分钟后相遇。

  小明的路程：70x
  小红的路程：50x

  70x + 50x = 840
       120x = 840
          x = 7
  ```
- 每步高亮对应线段图上的部分
- 最后 `x = 7` 用 YELLOW 放大强调
- 相遇点上出现 `7分钟` 标记

### Content
1. 屏幕分为左右：左 = 线段图，右 = 计算区域
2. 第一步：设未知数 x
3. 第二步：分别写出两人的路程表达式
4. 第三步：列出方程 70x + 50x = 840
5. 第四步：合并 120x = 840，解出 x = 7
6. 相遇点闪烁，出现"7分钟后相遇"
7. 两个小人最终在相遇点汇合

### Narration Notes
- 语速中等，每步停顿
- "我们设 x 分钟后相遇。那么小明走了 70x 米，小红走了 50x 米。"
- "两段加起来等于总路程 840 米。70x 加 50x 等于 120x，120x 等于 840..."
- 停顿——"所以 x 等于 7。7 分钟后，他们就见面啦！"

### Technical Notes
- 用 `TransformMatchingTex` 或逐步替换展示方程推导
- 每步用 `Indicate` 或颜色变化高亮当前项
- 最后 `x=7` 用 `FlashAround` 或缩放动画强调

---

## Scene 4: 解法二 — 巧算，速度和 × 时间
**Duration**: ~50s
**Purpose**: 展示第二种更"巧妙"的解法，为后续揭示做准备

### Visual Elements
- 线段图保留
- 两人图标同时开始向中间移动（动画演示每分钟两人靠近的过程）
- 屏幕出现思路提示：
  ```
  两人每分钟一共走：70 + 50 = 120 米
  ```
- 然后是方程：
  ```
  (70 + 50) × x = 840
         120 × x = 840
              x = 7
  ```
- 动画：两个小人每分钟同时向中间跳一段（120米），共跳7次到达相遇点

### Content
1. 回到线段图视角
2. 两人开始动画移动 — 每分钟两人同时移动
3. 强调："你看，每分钟两人一共靠近了 70+50 = 120 米"
4. 展示第二种列式方法：(70+50)x = 840
5. 同样得出 x=7
6. 两种解法的结果并排显示，都是 7

### Narration Notes
- 语气：发现新大陆的感觉
- "我们换个角度想：小明每分钟走70米，小红每分钟走50米。"
- "那两人每分钟一共走了... 70加50，等于120米！"
- "每分钟靠近120米，840米需要多少分钟？840除以120，还是7分钟！"

### Technical Notes
- 小人移动用 `animate.shift()` 或 `move_to()`
- 速度和中标注用 `MathTex`
- 解法二的公式与解法一上下排列，为下一场景的对比做铺垫

---

## Scene 5: 殊途同归 — 两种解法其实就是一回事
**Duration**: ~35s
**Purpose**: The "aha moment" — 揭示乘法分配律，让学生看到两种解法的内在联系

### Visual Elements
- 两种解法的方程上下排列：
  ```
  解法一：70x + 50x = 840
  解法二：(70 + 50)x = 840
  ```
- 解法一向解法二变形：`70x + 50x` → 黄色框框住 → `(70+50)x`
- 乘法分配律公式出现在中间：
  ```
  a×c + b×c = (a+b)×c
  ```
- `a=70`, `b=50`, `c=x` 分别对应高亮
- 最终两者重合，显示"完全一样！"

### Content
1. 两个方程并排显示
2. 上面的方程中，70x 和 50x 被框出
3. 显示乘法分配律公式
4. 把 a=70, b=50, c=x 代入，证明两者等价
5. 两个方程变换合并为一个
6. 出现 checkmark 或笑脸

### Narration Notes
- 语气：恍然大悟
- "等等，你有没有发现——70x + 50x 不就是 (70+50) 乘以 x 吗？"
- "这就是我们学的乘法分配律！a×c + b×c = (a+b)×c"
- "所以，解法一和解法二，其实是一回事。分开算也行，合起来算也行——都是对的！"

### Technical Notes
- `TransformMatchingTex` 是实现变形效果的关键
- 使用 `SurroundingRectangle` 框出对应项
- 颜色编码保持一致：BLUE = 小明相关，GREEN = 小红相关

---

## Scene 6: 总结 — 一个模型，一通百通
**Duration**: ~25s
**Purpose**: 总结核心模型，让学生带走最关键的一句话

### Visual Elements
- 所有内容退去
- 屏幕中央出现核心公式：
  ```
  速度和 × 相遇时间 = 总路程
  ```
- 公式中每个词用不同颜色标注
- 下方出现三个变式（小字）：
  ```
  相遇时间 = 总路程 ÷ 速度和
  速度和   = 总路程 ÷ 相遇时间
  甲路程 + 乙路程 = 总路程
  ```
- 最后出现结束画面

### Content
1. 核心公式从中心放大出现
2. 每个词逐个亮起
3. 三个变式公式从下方淡入
4. 画面优雅淡出

### Narration Notes
- 语气：总结、鼓励
- "所以，相遇问题的核心就一句话——速度和乘以时间等于总路程。"
- "记住这个模型，不管是走路、开车、还是工程队挖隧道，道理都一样！"
- "下次遇到相遇问题，你会先用线段图分析，再选自己喜欢的方法来算。"

### Technical Notes
- `MathTex` 展示公式
- `FadeIn` / `FadeOut` 处理场景过渡
- 结束画面用 `FadeToColor` 或直接淡出

## Transitions & Flow
- Scene 1→2: 道路图自然变形为抽象线段图（`ReplacementTransform`）
- Scene 2→3: 线段图收缩到左侧，右侧腾出计算空间
- Scene 3→4: 清空右侧计算，重新回到线段图全景，开始演示
- Scene 4→5: 两个方程上下排列，保持视觉连续性
- Scene 5→6: 方程收缩消失，核心公式从中央浮现
